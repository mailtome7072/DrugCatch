# 📌 프로젝트 플랜

## 1. 프로젝트 개요
- **목표:** 처방전 또는 약품 조제약 정보를 활용한 복약정보 조회 웹 MVP 개발
- **기간:** 2일 (Phase 1 완료 기준)
- **범위:** 이미지 입력 → 분석 → 결과 조회까지의 기본 흐름 구현

---

## 2. 핵심 기능
- 사용자 경고 및 동의 UI
- 이미지 입력 방식 선택 (촬영 vs 업로드)
- 이미지 미리보기 및 분석 시작 버튼
- OCR 기반 처방전 텍스트 추출
- 약품 이미지 인식 및 약품명 추출
- 데이터 파싱 및 구조화
- 분석 완료 후 결과 화면 (병명 → 증상 → 약품 카드)
- 에러 처리 및 로깅

---

## 3. 사용자 시나리오
1. **사용자 동의**
   - 사용자는 개인정보 처리 및 전문가 확인 안내를 읽고 동의 체크박스를 선택한다.
2. **이미지 입력**
   - 사용자는 처방전 또는 약품 사진을 촬영하거나 기기에서 업로드한다.
3. **미리보기**
   - 업로드된 이미지를 확인하고 필요 시 교체한다.
4. **분석 시작**
   - "이미지 분석 시작" 버튼을 클릭하면 OCR/이미지 인식이 진행된다.
   - 로딩 인디케이터가 표시된다.
5. **결과 확인**
   - 분석 완료 후 인디케이터 종료
   - 결과 화면에 **주요 병명 → 증상 → 약품 정보 카드** 순으로 표시된다.
   - 사용자에게 "최종 확인은 의사·약사에게"라는 안내가 제공된다.

---

## 4. 기술 스택
- **Frontend:** React.js / Next.js, TailwindCSS 또는 Material UI
- **Backend:** Node.js (Express), Python (OCR 모듈)
- **OCR/이미지 처리:** Tesseract OCR, OpenCV / TensorFlow Lite
- **CI/CD:** GitHub Actions (무료 티어 활용)
- **Infra:** Vercel/Netlify (무료 티어 활용)  
  ※ MVP 단계에서는 DB 없이 진행, 약품 데이터는 정적 JSON 또는 공공 API 활용

## 6. 프로젝트 단계별 계획

### Phase 0 — 프로젝트 초기 설정 ✅
- ✅ 저장소 생성 및 브랜치 전략 설정
- ✅ Claude Code 에이전트 설정
- ✅ CI/CD 파이프라인 구성
- ✅ 개발 프로세스 문서화
- ✅ Git 협업 규칙 문서화 (docs/git-convention.md)
- ✅ 환경 설정 가이드 완성 (docs/setup-guide.md)

### Phase 1 — 파일럿 MVP 바이브 코딩

#### Sprint 1 (Day 1)
- 사용자 경고 및 동의 UI 레이아웃 구성
- 개인정보 및 민감 정보 처리 안내
- 전문가 확인 안내
- 동의 체크박스 및 확인 버튼
- 동의 완료 후 이미지 입력 화면 전환 (라우팅)

#### Sprint 2 (Day 1)
- 이미지 입력 방식 선택 (촬영 vs 업로드)
- 업로드/촬영된 이미지 미리보기
- 이미지 재선택/교체 기능
- "이미지 분석 시작" 버튼 (로딩 인디케이터 포함)
- 에러 처리 및 로깅
- 내부 테스트 및 피드백 반영

#### Sprint 3 (Day 2)
- 업로드된 이미지 유형 판별 (처방전 vs 조제약 사진)
- OCR 처리 및 약품명 추출 (Tesseract OCR)
- 약품 이미지 인식 모듈 구현 (OpenCV/TensorFlow Lite)
- 데이터 파싱 및 구조화
- 정적 JSON 또는 공공 API로 약품 데이터 조회
- 분석 완료 후 결과 응답 반환
- 에러 처리 및 로깅 (OCR 실패, 파싱 오류 등)

#### Sprint 4 (Day 2)
- "이미지 분석 시작" 버튼과 백엔드 OCR/이미지 인식 API 연결
- 조회 결과 UI 레이아웃 구성
  - 최상단: 유추된 주요 병명 표시
  - 중간: 처방 목적 주요 증상
  - 하단: 약품 정보 카드 리스트
- 에러 처리 및 로깅
- 내부 테스트 및 피드백 반영
